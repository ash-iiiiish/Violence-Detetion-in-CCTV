1️⃣ Real CCTV Systems Do NOT Process Entire Videos

    They process frames as a stream, not full videos.

    Instead of:

        Video → process entire file → output

    Real systems do:

        Camera stream → frame → prediction → next frame

    Example pipeline:

        CCTV Stream
            ↓
        Frame Capture (every 0.5–1 sec)
            ↓
        Detection Model
            ↓
        Violence Classifier
            ↓
        Alert System

    So the model only processes small frame batches, not whole videos.

2️⃣ Frame Skipping (Huge Optimization)

    You don’t need 30 FPS for violence detection.

    Typical CCTV AI systems process:

    1–5 frames per second

    Example:

        30 FPS camera
        ↓
        Process only 3 FPS
        ↓
        90% computation reduced

    This alone can speed your system 10×.

3️⃣ Sliding Window for Video Models

    Your R3D-18 requires multiple frames.

    Instead of waiting for a whole video:

    Example:

        Collect 16 frames
        ↓
        Run violence classifier
        ↓
        Slide window

    Pipeline:

        Frame1-16 → prediction
        Frame2-17 → prediction
        Frame3-18 → prediction

    This allows continuous real-time prediction.

4️⃣ Run Detection Only When Needed

    You can reduce computation by using trigger logic.

    Example:

        Person detected?
        ↓
        Yes → Run violence model
        No → Skip processing

    Person detection using YOLOv8 is very fast.

    So violence detection runs only when people are present.
